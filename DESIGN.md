# SPAL Design Operating System
Version 2.0

> **This is the single source of truth for all visual decisions.**
> Product strategy, audience, and engineering rules live in CLAUDE.md.
> Every screen, component, motion, and illustration decision should be checked against this document first.

---

## Why This Document Exists

SPAL is evolving from a functional MVP into a premium product.

The current product works. The problem is perception.

Users can immediately tell parts of the experience were generated by AI tools because the design follows common patterns:

- Generic fintech color usage
- Over-reliance on rounded cards
- Standard SaaS layouts
- Generic iconography
- Emoji-style visual elements
- Multiple competing accent colors
- No distinctive visual language
- No recognizable brand assets
- No proprietary illustration system
- No memorable graphic system

The current interface feels like: **"A fintech app."**

Instead of: **"SPAL."**

Every design decision should move SPAL away from looking like a template and toward looking like a crafted product.

---

## Design Goal

SPAL should feel:
- Calm
- Intelligent
- Progressive
- Trustworthy
- Premium
- Human

SPAL should NOT feel:
- Corporate
- Generic
- Gamified
- Childish
- Overly playful
- Like an AI-generated startup template

---

## Visual Positioning

SPAL sits between **Linear**, **Arc Browser**, **Amigo**, and **Fruitful** — modern editorial products — while remaining approachable for small business owners.

The feeling should be: **"Professional financial intelligence made simple."**

Not: "Fun business app."

---

## Core Design Philosophy

The product should communicate confidence through clarity.

- Do not decorate unnecessarily.
- Do not add visual elements because they look cool.
- Every element should communicate: growth, understanding, progress, confidence.

**Clarity beats decoration. Identity beats trends. Systems beat aesthetics.**

## Unique Visual Territory

SPAL does not use mascots.
SPAL does not rely on generic fintech illustrations.
SPAL communicates through a **proprietary system of geometric symbols**.

These symbols represent business concepts. The symbol system is a core part of the brand.

**Symbol vocabulary:**
| Concept | Visual Form |
|---|---|
| Flow | Orbital motion |
| Growth | Ascending layers |
| Profit | Expanding rings |
| Opportunity | Opening forms |
| Focus | Concentric circles |
| Consistency | Repeating geometry |
| Momentum | Directional geometry |
| Insight | Radiating structure |
| Scale | Expanding structures |
| Cashflow | Continuous waves |
| Loop | Orbital motion |

Symbols should appear across: onboarding, insights, goals, reports, learning experiences, marketing, empty states.

Users should eventually recognize SPAL through its symbols alone.

---

## Symbol System

**Inspired by:** geometric abstraction, motion, systems thinking, modern operating systems.

**Visual characteristics:**
- Soft gradients
- Translucent layers
- Geometric construction
- Mathematical balance
- Subtle depth
- Minimal visual noise

**References:** Arc Browser, Linear graphics, modern AI visual systems.

**Avoid:** stock illustrations, cartoon characters, generic 3D renders, emoji-based design.

## Illustration Direction

Illustrations should not tell stories through characters.
Illustrations should **communicate concepts**.

They should feel: intelligent, timeless, premium.

---

## Icon System

Icons should be **custom**. Never use generic icon libraries directly.

**Characteristics:**
- Geometric
- Consistent stroke width
- Rounded terminals
- Minimal detail
- Built on a shared grid

**Icons should feel closer to:** Arc, Raycast, Linear

**Not:** Material Icons, Emoji, Generic fintech icons

---

## Motion System

Motion is a **primary brand asset**.

Everything should feel like **flow**. Not bounce. Not spring. Not playful.

Motion should feel: fluid, intentional, elegant.

**Examples:**
- Profit increases smoothly
- Graphs grow organically
- Insights fade upward
- Cards expand naturally
- Numbers animate softly
- Progress indicators move continuously
- Loop concepts rotate subtly

**Framer Motion easing:** Prefer `easeOut`, `easeInOut`, and custom cubic beziers. Never use `spring` with high stiffness/bounce values.

**Avoid:** excessive bounce, exaggerated springs, flashy effects.

## Typography

Typography is one of SPAL's strongest brand assets. The interface should rely heavily on hierarchy.

**Typefaces:**
| Role | Typeface |
|---|---|
| Primary | **Satoshi** |
| Alternative | **Geist** |
| Secondary | **Inter Tight** |
| ❌ Avoid | **Poppins** — heavily associated with generic startup and AI-generated products |

**Typography Principles:**
- Large headings
- Clear hierarchy
- Generous spacing
- Strong contrast
- Minimal visual clutter
- Numbers should feel important
- Financial figures should command attention

**Scale:**
| Token | Size | Weight |
|---|---|---|
| Display | 36–40px | Bold / ExtraBold |
| H1 | 32px | Bold |
| H2 | 24px | SemiBold |
| H3 | 20px | SemiBold |
| H4 | 18px | Medium |
| Body | 14–16px | Regular |
| Small | 12px | Regular |
| Label | 11px | Medium |

## Color Philosophy

**Current issue:** Too many colors compete equally. This creates visual noise.

**New system:**

| Role | Name | Hex | Usage |
|---|---|---|---|
| Primary | Midnight Navy | `#0F172A` | Text, surfaces, dominant tone |
| Surface | Off White | `#F8F7F4` | Page background |
| Accent | SPAL Green | `#22C55E` | Sales, success, primary CTA |
| Secondary | SPAL Blue | `#2563EB` | Profit, AI, insights |
| Support | SPAL Purple | `#8B5CF6` | Learn/Coach, growth |
| Action | SPAL Orange | `#FF7A00` | Expenses only — sparingly |

**Rules:**
- Navy dominates. It is the ground tone.
- Green is the primary accent — one confident color.
- Orange is an **action color**, not a brand color. Use it only to indicate expenses/money out.
- Do not let all four colors appear at equal visual weight on a single screen.

## Layout Philosophy

The current product is card-heavy. Cards are useful but should not define the experience.

**Future layouts should feel:** spacious, editorial, intentional.

**Use:**
- Larger sections
- Stronger typography
- More breathing room
- Clearer hierarchy

**Inspired by:** Amigo, Fruitful, Without Studio.

**Avoid:** nested cards inside cards, cluttered action rows, equally-weighted UI sections.

## Component Rules

### Cards
- Use sparingly — cards should not define the experience
- When used: subtle border (`border border-black/6`), minimal shadow
- Prefer borderless sections with generous padding over card stacking

### Buttons
- Primary: Navy fill or Green fill depending on context
- Rounded-full only
- Minimum 48px height (touch target)
- No gradient on primary buttons — solid fills only
- Text: always use direct `style={}` or ensure `!important` overrides work (gradient `background-image` cannot be overridden by `bg-*` Tailwind classes alone)

### Bottom Sheets
- Preferred over modals
- Drag handle always visible
- Backdrop: `rgba(0,0,0,0.4)` — not too dark

### Empty States
- Use a SPAL geometric symbol (SVG) — never emoji
- One line of context, one suggested action
- Feel purposeful, not apologetic

### Loading States
- Skeleton screens, not spinners
- Match the shape of the content being loaded

## What "SPAL" Looks Like in Code

When generating any screen, component, or UI piece, ask:

1. **Does this use a SPAL geometric symbol instead of an emoji or stock icon?**
2. **Is Satoshi/Geist used instead of Poppins for headings?**
3. **Does orange appear only for expenses/money-out contexts?**
4. **Is motion fluid (easeOut) rather than bouncy (spring)?**
5. **Is the layout spacious and editorial, not card-stacked?**
6. **Does every element communicate growth, understanding, progress, or confidence?**

If the answer to any of these is no — revise before shipping.

---

## Product Experience Principles

Every screen should answer one question:

**"Does this help the user understand and improve their business?"**

If not, remove it.

- Clarity beats decoration
- Identity beats trends
- Systems beat aesthetics

SPAL should feel like a **business operating system for everyday entrepreneurs**. Not another AI app.
